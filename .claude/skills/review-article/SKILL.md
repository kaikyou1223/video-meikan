# 記事品質チェックスキル

記事を公開前にレビューし、品質を担保する。自動チェック + 手動確認の2段階。

---

## Step 1: 自動バリデーション実行

バリデーションスクリプトを実行する:

```bash
php meikan/batch/validate_articles.php
```

**チェック項目:**
- `@actress[slug]` の参照先がDBに存在するか
- 画像URLの形式チェック（NOW PRINTING検出）
- FANZA CIDのDB存在チェック
- CID数字プレフィックス検出（DMMの新URL形式で404になるため自動補正される）
- slug形式の正当性
- **CID-女優整合性チェック**: 各`## 女優名`セクション内のCID（画像URL・FANZAリンク・サンプル画像）が、そのセクションの`@actress[slug]`の女優に`actress_work`で紐付いているか検証。数字プレフィックス付きCID（例: `1ipzz00780`）はプレフィックスなし版（`ipzz00780`）でも照合する

エラーがあればまず修正してから次のステップへ進む。

---

## Step 2: 記事構造チェック（手動確認）

対象記事を読み込み、以下を確認:

### 2-1. フロントマター
- [ ] `title` が設定されている
- [ ] `description` がSEO向けに120文字以内で書かれている
- [ ] `date` が正しいフォーマット（YYYY-MM-DD）
- [ ] `slug` がURL-safe（小文字英数字+ハイフン）

### 2-2. 記事本文
- [ ] H2見出しが3つ以上ある（目次が自動生成される条件）
- [ ] 導入文（最初のH2の前）がある
- [ ] `@actress[slug]` が全て存在するslugを参照している（Step1で検証済み）
- [ ] 画像が適切に挿入されている（NOW PRINTINGでない）
- [ ] `:::say` ブロック（AV博士コメント）が適度に配置されている
- [ ] 内部リンク・外部リンクが適切に設定されている
- [ ] 旧WordPress形式の会話ブロックが残っていないか（**全記事対象**）
  - `![名前](url)` + 名前テキスト + メッセージ の繰り返しパターンはアバター画像が全幅表示になる
  - 検出: `grep -n '!\[編集部\|!\[筆者\|!\[ヒラタ\|!\[タナカ\|!\[ヤマシタ' ファイル名`
  - 修正: `:::chat` ブロックに変換する（`名前[avatar_url]: メッセージ` 形式）
- [ ] `:::chat` ブロックを使っている場合、スピーカー行の形式を確認
  - `名前: メッセージ` — 問題なし
  - `名前[https://...]: メッセージ` — アバターURL付き形式。パーサーのバグが修正済みのため現在は正常動作するが、念のためブラウザで吹き出し表示を確認
  - 検出コマンド: `grep -n '^\w.*\[https\?://' ファイル名`（`:::chat` ブロック内の行に限定）

### 2-5. 移行記事の追加チェック（本番からの移行記事の場合のみ）

- [ ] **titleが本番と一致しているか**: 本番HTMLのH1/meta titleと `title:` フィールドを照合する（移行スクリプトの自動生成値がズレることがある）
- [ ] **空alt画像 `![](url)` が残っていないか**: カスタムパーサーは空altを認識せずテキスト表示する。`grep -c '!\[\](' ファイル名` でカウント → 0であること。見つかったら `!img[url]` に変換
- [ ] **会話ブロック `![名前](url)` + テキスト形式が残っていないか**: 吹き出し会話が未変換のままだとアバター画像が全幅で表示される。`grep -n '!\[編集部\|!\[筆者\|!\[フェチ' ファイル名` で検出 → `:::chat` に変換
- [ ] **author bio（筆者プロフィール）が残っていないか**: `![この記事の筆者](...)` や `AV評論家ヤマシタ` + プロフィール文のブロックは除外すべき要素。`grep -n 'この記事の\|AV評論家ヤマシタ' ファイル名` で確認

### 2-3. 画像カルーセル
- [ ] 3枚以上連続する `!img[url]` が `:::samples` ブロックに変換されているか
  - 検出: `grep -n '^!img\[' ファイル名` で連続3行以上の箇所を確認
  - 修正: 連続する `!img[` をまとめて `:::samples` ... `:::` で囲む（間の空行は除去）
  - 例外: `:::cast` など他ブロック内の `!img[` はスキップ

### 2-4. SEO
- [ ] タイトルにメインキーワードが含まれている
- [ ] メタディスクリプションにキーワードが含まれている
- [ ] H2にサブキーワードが含まれている
- [ ] 画像のalt属性が適切に設定されている

## Step 3: 画像URL全件HTTPチェック

記事内の全画像URLに対してHTTPリクエストを送り、アクセス可能か確認する:

```bash
# 記事内の画像URLを抽出してHTTPステータスを確認
grep -oP '!\[.*?\]\(\K[^)]+' meikan/content/articles/対象記事.md | while read url; do
  status=$(curl -o /dev/null -s -w "%{http_code}" -L "$url")
  echo "$status $url"
done
```

- **200**: OK
- **403/404**: 画像が削除されている → 代替画像に差し替え
- **タイムアウト**: CDN側の問題の可能性 → 再確認

---

## Step 4: ブラウザ確認（任意）

代表的な記事ページをブラウザで開き、以下を目視確認:
- レイアウト崩れがないか
- 女優カードが正しく表示されているか
- 作品埋め込みが正しく表示されているか

agent-browserスキルを使って自動確認も可能:
```
/agent-browser で対象ページを開いてスクリーンショットを撮る
```
