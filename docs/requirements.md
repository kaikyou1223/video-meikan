# video-meikan 要件定義書

## 1. プロジェクト概要

### 1.1 サービス名
video-meikan（動画名鑑）

### 1.2 コンセプト
AV女優×ジャンルの組み合わせに特化したデータベースサイト。
「女優名 ジャンル」で検索するユーザーに最適化されたページを提供し、検索上位表示を目指す。

### 1.3 ターゲットユーザー
- 30〜40代の男性
- 「瀬戸環奈 NTR」のようなクエリで検索するユーザー

### 1.4 参考サイト
- https://av-wiki.net/av-actress/yukina-mafuyu/

---

## 2. サイト構成

### 2.1 ディレクトリ構成

```
/ (ルート)
├── [記事ディレクトリ]  ← WordPress（既存・別運用）
└── /meikan/             ← 本プロジェクト（PHP独自実装）
    ├── index.php        ← 名鑑TOP（女優一覧）
    ├── /{actress_slug}/ ← 女優個別ページ
    │   ├── index.php    ← ジャンル一覧＋作品カルーセル
    │   └── /{genre_slug}/ ← ジャンル別作品一覧ページ
    └── /genre/          ← ジャンル一覧ページ（任意）
```

### 2.2 ページ一覧

| ページ | URL | 概要 |
|--------|-----|------|
| 名鑑TOP | `/meikan/` | 女優一覧。名前・サムネイル・作品数を表示 |
| 女優ページ | `/meikan/{actress}/` | ジャンル一覧＋ジャンルごとの作品カルーセル |
| ジャンル別ページ | `/meikan/{actress}/{genre}/` | 該当女優×ジャンルの作品一覧（個別表示） |

## 3. 機能要件

### 3.1 名鑑TOPページ (`/meikan/`)

- 登録女優の一覧を表示
- 各女優のサムネイル、名前、作品数を表示
- 女優個別ページへのリンク

### 3.2 女優個別ページ (`/meikan/{actress}/`)

- 女優のプロフィール情報（名前、画像）
- ジャンル一覧（該当女優の作品が存在するジャンルのみ表示）
- ジャンルごとの作品カルーセル（横スクロール）
  - 各作品はサムネイル表示
  - クリックでアフィリエイトリンクへ遷移

### 3.3 ジャンル別作品ページ (`/meikan/{actress}/{genre}/`)

- 該当女優×ジャンルの作品を個別に一覧表示
- 各作品の表示項目：
  - **タイトル**
  - **サムネイル画像**
  - **発売日**
  - **レーベル**
  - **リンク先**（アフィリエイトリンク）
- ページネーション（作品数が多い場合）

## 4. データ設計

### 4.1 データソース

| ソース | 取得方法 | 優先度 |
|--------|----------|--------|
| FANZA (DMM) | FANZA API（アフィリエイトAPI） | 主 |
| MGS動画 | API（あれば）またはスクレイピング | 副 |

### 4.2 データモデル

#### actresses（女優テーブル）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INT (PK) | 自動採番 |
| name | VARCHAR(100) | 女優名 |
| slug | VARCHAR(100) | URL用スラッグ |
| thumbnail_url | VARCHAR(500) | サムネイルURL |
| created_at | DATETIME | 登録日時 |
| updated_at | DATETIME | 更新日時 |

#### genres（ジャンルテーブル）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INT (PK) | 自動採番 |
| name | VARCHAR(100) | ジャンル名 |
| slug | VARCHAR(100) | URL用スラッグ |
| fanza_genre_id | VARCHAR(50) | FANZA側のジャンルID |

#### works（作品テーブル）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INT (PK) | 自動採番 |
| title | VARCHAR(500) | 作品タイトル |
| thumbnail_url | VARCHAR(500) | サムネイルURL |
| release_date | DATE | 発売日 |
| label | VARCHAR(200) | レーベル名 |
| affiliate_url | VARCHAR(1000) | アフィリエイトリンク |
| source | ENUM('fanza','mgs') | データソース |
| source_id | VARCHAR(100) | ソース側の作品ID |
| created_at | DATETIME | 登録日時 |
| updated_at | DATETIME | 更新日時 |

#### actress_work（女優×作品 中間テーブル）

| カラム | 型 | 説明 |
|--------|-----|------|
| actress_id | INT (FK) | 女優ID |
| work_id | INT (FK) | 作品ID |

#### work_genre（作品×ジャンル 中間テーブル）

| カラム | 型 | 説明 |
|--------|-----|------|
| work_id | INT (FK) | 作品ID |
| genre_id | INT (FK) | ジャンルID |

## 5. 技術構成

### 5.1 技術スタック

| 項目 | 技術 |
|------|------|
| 言語 | PHP 8.x |
| DB | MySQL（WordPress既存DB共用 or 別DB） |
| Webサーバー | Apache（WordPress環境に準拠） |
| フロント | HTML + CSS + vanilla JS（カルーセル用） |
| データ取得 | FANZA API / MGSスクレイピング（CLIバッチ） |

### 5.2 PHPを採用する理由と注意点

**メリット：**
- 既存WordPressと同一サーバーで運用可能
- レンタルサーバーでも動作する
- デプロイが容易（FTPアップロード）

**注意点：**
- SEO重視のため、SSR（サーバーサイドレンダリング）で完結させる（SPAにしない）
- URL構造は `.htaccess` の `RewriteRule` で制御
- テンプレートエンジンは素のPHP or 軽量テンプレート（Blade standalone等）

### 5.3 ディレクトリ構成（実装）

```
/meikan/
├── index.php              ← エントリーポイント（ルーター）
├── .htaccess              ← URL書き換え
├── config/
│   └── database.php       ← DB接続設定
├── src/
│   ├── Router.php         ← URLルーティング
│   ├── Database.php       ← DB接続クラス
│   ├── models/
│   │   ├── Actress.php
```

## 6. SEO要件

### 6.1 URL設計
- **女優ページ**: `/meikan/seto-kanna/`
- **ジャンル別**: `/meikan/seto-kanna/ntr/`
- クリーンURL（パラメータなし）

### 6.2 メタ情報
- 各ページに固有の `<title>` / `<meta description>` を設定
- 例：`<title>瀬戸環奈のNTR作品一覧 | video-meikan</title>`

### 6.3 構造化データ
- JSON-LD で ItemList / BreadcrumbList を出力

### 6.4 内部リンク
- パンくずリスト: TOP > 女優名 > ジャンル名
- 女優ページ間・ジャンル間の相互リンク

### 6.5 サイトマップ
- `/meikan/sitemap.xml` を動的生成

## 8. データ取得バッチ

### 8.1 FANZA API バッチ (`batch/fetch_fanza.php`)
- FANZA アフィリエイト API を使用
- 女優名で検索し、作品情報を取得
- ジャンル情報も API レスポンスから取得
- 定期実行（cron）で新作を自動取得

### 8.2 MGS バッチ (`batch/fetch_mgs.php`)
- API があれば API 経由、なければスクレイピング
- FANZA と同じデータ構造に正規化して DB に格納

### 8.3 実行頻度
- 日次または週次で cron 実行
- 初回は女優ごとに全件取得、以降は差分取得

## 10. 非機能要件

### 10.1 パフォーマンス
- ページ表示速度: 2秒以内（目標）
- 画像は外部URL（FANZA/MGS）を使用、遅延読み込み（lazy loading）

### 10.2 キャッシュ
- ページキャッシュ（ファイルキャッシュ or APCu）を導入
- データ更新バッチ実行時にキャッシュクリア

### 10.3 セキュリティ
- SQLインジェクション対策（PDOプリペアドステートメント）
- XSS対策（出力エスケープ）
- `.htaccess` で `batch/` ディレクトリへの外部アクセスを遮断

### 10.4 運用
- エラーログの出力
- バッチ実行ログの記録
